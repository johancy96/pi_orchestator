# Master Orchestrator: UI/UX & Design System Standards

This skill defines the standards for creating accessible, intuitive, and visually consistent user interfaces. As a Master Orchestrator, you must ensure that every UI component is user-centric and follows professional design principles.

## 1. UI/UX Principles
- **User-Centric Design**: Prioritize the user's needs and mental models. The interface should be intuitive and require minimal cognitive load.
- **Visual Hierarchy**: Use size, color, and spacing to guide the user's eye to the most important actions.
- **Consistency**: Maintain consistent patterns (buttons, inputs, layouts) throughout the application to build user trust.
- **Feedback & States**: Provide immediate feedback for user actions (loading states, success/error messages, hover effects).

## 2. Accessibility (WCAG)
- **Perceivable**: Ensure content is accessible to all senses (e.g., alt text for images, high color contrast).
- **Operable**: The interface must be navigable via keyboard and assistive technologies.
- **Understandable**: Use clear language and predictable navigation patterns.
- **Robust**: Ensure the UI works across different browsers, devices, and screen readers.
- **Standards Compliance**: Aim for **WCAG 2.1 AA** compliance as the baseline for all professional projects.

## 3. Design Systems & Component Libraries
- **Atomic Design**: Build interfaces starting from atoms (buttons, labels) to organisms (headers, forms) to full pages.
- **Theming & Design Tokens**: Use design tokens (colors, typography, spacing) to ensure consistency and facilitate global theme changes.
- **Component Reusability**: Build modular, stateless components that can be easily reused across the application.
- **Responsive Design**: Use modern techniques (Flexbox, Grid, Media Queries) to ensure the UI looks great on any screen size.

## 4. Master Orchestrator Protocol
- **UX Validation**: During the planning phase, define the "Happy Path" for the user and ensure the architecture supports it efficiently.
- **Accessibility Audit**: Before marking a UI task as `[x]`, perform a basic accessibility check (keyboard navigation, contrast ratio).
- **Design Alignment**: Ensure all UI implementations follow the project's design system or established visual standards.
